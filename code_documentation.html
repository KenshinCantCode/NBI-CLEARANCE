1. Project Overview

The NBI Clearance Portal is a secure web application that manages NBI (National Bureau of Investigation) clearance applications with the following features:

  • User Registration and Login with OTP verification
  • Admin Backend Dashboard
  • Application Management
  • Appointment Scheduling
  • Notifications System
  • Email Integration
  • Password Recovery
  • Role-based Access Control (User vs Admin)

Technology Stack

Component Technology
Backend PHP with MySQLi
Database MySQL
Frontend HTML, CSS, JavaScript
Email PHPMailer
Security Prepared Statements, Password Hashing (PASSWORD_DEFAULT), OTP Tokens

3. Authentication & Helpers (auth_helpers.php)

Functions

A. Session Management

  • setFlashMessage(string $type, string $message): void
    • Stores a temporary message in session for display on next page load
    • Types: 'success', 'error', 'info', 'warning'
    • Used for showing status after redirects
  • pullFlashMessage(): ?array
    • Retrieves and removes flash message from session
    • Returns array with 'type' and 'message', or null if none exists
    • Auto-clears after retrieval (one-time display)

B. Navigation

  • redirectTo(string $location): void
    • Sends HTTP redirect header to specified location
    • Terminates execution to prevent further code running
    • Used: After login success, form submissions, error conditions

C. Configuration

  • loadAppConfig(): array
    • Loads application configuration from app_config.php file
    • Returns empty array if config file doesn't exist
    • Uses static caching to load file only once per request
  • configuredAppBaseUrl(): string
    • Returns base URL from environment variables or config file
    • Checks in this order: PUBLIC_BASE_URL, APP_BASE_URL, APP_URL, config
    • Ensures URL starts with http:// or https://

D. Password Security

  • isLegacyMd5Hash(string $storedHash): bool
    • Checks if hash is old MD5 format (32 hex characters)
    • Returns true if matches pattern /^[a-f0-9]{32}$/i
    • Used to identify passwords needing upgrade
  • verifyPasswordAndUpgrade(mysqli $conn, string $email, string $plainPassword, string $storedHash): bool
    • Automatically migrates legacy passwords to secure bcrypt
    • Uses hash_equals() to prevent timing attacks
    • New passwords always stored as bcrypt

E. Token Generation

  • createToken(int $lengthBytes = 32): string
    • Generates random token using random_bytes()
    • Default: 32 bytes = 64 hex characters
    • Uses: Password reset tokens, OTP generation
    • Cryptographically secure randomness
  • hashToken(string $token): string
    • Hashes token with SHA256
    • Returns 64-character hex string
    • DATABASE STORAGE: Only hash is stored, never plain token

F. User Roles

  • ensureUserRoleColumn(mysqli $conn): void
    • Adds 'role' column to users table if missing
    • Creates column: role VARCHAR(20) NOT NULL DEFAULT 'user'
    • Useful if upgrading from older database version
  • getUserRole(mysqli $conn, string $email): string
    • Retrieves role for given email
    • Returns: 'admin' or 'user' (default)
  • isAdminUser(mysqli $conn, string $email): bool
    • Convenience function checking if user is admin
    • Returns true if role === 'admin'

4. User Registration & Login (register.php)

Purpose

Handles user sign-up, sign-in, and OTP verification initiation

Key Functions

A. OTP Email Generation

  • createLoginOtpHash(string $email, string $otp): string
    • Creates hash from email + "|" + otp
    • Uses hashToken() which calls sha256()
    • Example: createLoginOtpHash("user@email.com", "123456") = sha256("user@email.com|123456")
  • sendLoginOtpEmail(mysqli $conn, string $email, string $firstName, string $lastName): array
    • Invalidates old unused OTPs for this email
    • Generates new 6-digit OTP
    • Stores hash in database with 10-minute expiration
    • Sends formatted email with OTP

B. User Sign-Up

$firstName = trim($_POST['fName'] ?? ''); // Get & trim first name
$lastName = trim($_POST['lName'] ?? ''); // Get & trim last name
$email = trim($_POST['email'] ?? ''); // Get & trim email
$password = $_POST['password'] ?? ''; // Get password (no trim)

// Validate all fields present
if ($firstName === '' || $lastName === '' ||
    !filter_var($email, FILTER_VALIDATE_EMAIL) || $password === '') {
    setFlashMessage('error', 'Please complete all sign up fields.');
    redirectTo('index.php?form=signup');
}

// Check password length
if (strlen($password) < 8) {
    setFlashMessage('error', 'Password must be at least 8 characters long.');
    redirectTo('index.php?form=signup');
}

C. User Sign-In

// Generate OTP and send email
$mailResult = sendLoginOtpEmail($conn, $user['email'],
                      (string)($user['firstName'] ?? ''),
                      (string)($user['lastName'] ?? ''));

if (!$mailResult['success']) {
    // Email sending failed
    setFlashMessage('error', 'Unable to send OTP email. ' . $mailResult['error']);
    redirectTo('index.php');
}

// Store email in session for OTP verification page
session_regenerate_id(true); // Prevent session fixation
$_SESSION['pending_login_email'] = $user['email'];

redirectTo('verify_login.php');

5. Login Verification (verify_login.php)

Purpose

OTP verification interface and logic

OTP Verification Process

  1. Check pending email exists
  2. Extract 6-digit code from user input
  3. Fetch OTP record from database
  4. Validate OTP not expired
  5. Verify OTP using hash comparison
  6. Mark OTP as used (prevent replay attacks)
  7. Create authenticated session
  8. Redirect to dashboard

Security Features

  • OTP never stored in plain text, only SHA256 hash
  • hash_equals() prevents timing attacks
  • 10-minute expiration prevents brute force
  • Single-use tokens (marked used after verification)
  • Session regeneration prevents session fixation
  • Cannot reuse expired OTPs

6. Password Recovery (forgot_password.php)

Purpose

Initiate password reset by sending email with reset link

Process

  1. Validate email format and check if user exists
  2. Invalidate old unused reset tokens
  3. Generate new reset token (32 bytes = 64 chars)
  4. Hash token with SHA256
  5. Store hash in database with 30-minute expiration
  6. Send email with plain token in URL
  7. Show success message (security: same message whether email found or not)

Database Table (password_resets)

Column Type Description
email VARCHAR(255) User's email
token_hash VARCHAR(64) SHA256 of token
expires_at TIMESTAMP 30 minutes from generation
used_at TIMESTAMP NULL Set when used to prevent reuse
created_at TIMESTAMP When created

7. Password Reset (reset_password.php)

Purpose

Handle password reset with token validation

Validation Process

  1. Hash token and lookup in password_resets table
  2. Validate token not used, not expired
  3. Show password reset form if valid
  4. Hash new password with password_hash()
  5. Update users table password
  6. Mark token as used
  7. Revoke all pending OTP logins for security
  8. Redirect to login with success message

Security Features

  • Token validated and hashed before display
  • Single-use tokens (checked and marked as used)
  • 30-minute expiration
  • All pending logins revoked when password reset (forces re-login)
  • Password validation requirements enforced

Data Flow Summary

Registration Flow

  1. User → index.php (Sign Up form)
  2. Form POST → register.php ($_POST['signUp'])
  3. Validate fields and check email uniqueness
  4. Hash password with password_hash()
  5. Insert into users table
  6. Flash message and redirect to index.php

Sign In Flow

  1. User → index.php (Sign In form)
  2. Form POST → register.php ($_POST['signIn'])
  3. Validate email format and check password not empty
  4. Query users table for matching email
  5. Verify password with verifyPasswordAndUpgrade()
  6. Call sendLoginOtpEmail()
  7. Store email in session
  8. Redirect to verify_login.php

OTP Verification Flow

  1. User receives OTP email
  2. User enters OTP on verify_login.php
  3. Form POST → verify_login.php ($_POST['verify'])
  4. Query login_verifications table
  5. Validate OTP not expired
  6. Verify OTP using hash comparison
  7. Mark as used, create session, redirect to homepage
