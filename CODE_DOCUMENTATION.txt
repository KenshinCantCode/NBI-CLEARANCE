1. PROJECT OVERVIEW
================================================================================

NBI Clearance Portal is a secure web application that manages NBI (National 
Bureau of Investigation) clearance applications with the following features:

- User Registration and Login with OTP verification
- Admin Backend Dashboard
- Application Management
- Appointment Scheduling
- Notifications System
- Email Integration
- Password Recovery
- Role-based Access Control (User vs Admin)

Technology Stack:
- Backend: PHP with MySQLi
- Database: MySQL
- Frontend: HTML, CSS, JavaScript
- Email: PHPMailer
- Security: Prepared Statements, Password Hashing (PASSWORD_DEFAULT), OTP Tokens

3. AUTHENTICATION & HELPERS (auth_helpers.php)
================================================================================

PURPOSE: Provides core authentication functions used throughout the application

FUNCTIONS:

A. SESSION MANAGEMENT:

   setFlashMessage(string $type, string $message): void
   - Stores a temporary message in session for display on next page load
   - Types: 'success', 'error', 'info', 'warning'
   - Used for showing status after redirects
   
   pullFlashMessage(): ?array
   - Retrieves and removes flash message from session
   - Returns array with 'type' and 'message', or null if none exists
   - Auto-clears after retrieval (one-time display)

B. NAVIGATION:

   redirectTo(string $location): void
   - Sends HTTP redirect header to specified location
   - Terminates execution to prevent further code running
   - Used: After login success, form submissions, error conditions

C. CONFIGURATION:

   loadAppConfig(): array
   - Loads application configuration from app_config.php file
   - Returns empty array if config file doesn't exist
   - Uses static caching to load file only once per request
   
   configuredAppBaseUrl(): string
   - Returns base URL from environment variables or config file
   - Checks in this order: PUBLIC_BASE_URL, APP_BASE_URL, APP_URL, config
   - Ensures URL starts with http:// or https://
   
   appBaseUrl(): string
   - Fallback method to detect base URL from server variables
   - Detects HTTPS from: HTTP_X_FORWARDED_PROTO, HTTPS, SERVER_PORT
   - Gets host from: HTTP_X_FORWARDED_HOST, HTTP_HOST, SERVER_NAME
   - Constructs full base URL scheme://host/basepath

D. PASSWORD SECURITY:

   isLegacyMd5Hash(string $storedHash): bool
   - Checks if hash is old MD5 format (32 hex characters)
   - Returns true if matches pattern /^[a-f0-9]{32}$/i
   - Used to identify passwords needing upgrade
   
   verifyPasswordAndUpgrade(mysqli $conn, string $email, string $plainPassword, string $storedHash): bool
   
   PROCESS:
   1. First attempts PHP's password_verify() with bcrypt hash
   2. If fails and old MD5 hash detected:
      - Compares with md5($plainPassword) using hash_equals() (timing-safe)
      - If match: upgrades to bcrypt (PASSWORD_DEFAULT) in database
   3. Returns true if password matches, false otherwise
   
   SECURITY BENEFITS:
   - Automatically migrates legacy passwords to secure bcrypt
   - Uses hash_equals() to prevent timing attacks
   - New passwords always stored as bcrypt

E. TOKEN GENERATION:

   createToken(int $lengthBytes = 32): string
   - Generates random token using random_bytes()
   - Default: 32 bytes = 64 hex characters
   - Uses: Password reset tokens, OTP generation
   - Cryptographically secure randomness
   
   hashToken(string $token): string
   - Hashes token with SHA256
   - Returns 64-character hex string
   - DATABASE STORAGE: Only hash is stored, never plain token
   - For OTP: hash = sha256(email + "|" + otp)

F. USER ROLES:

   ensureUserRoleColumn(mysqli $conn): void
   
   FUNCTION: Adds 'role' column to users table if missing
   - Executes only once per request (static $checked)
   - Creates column: role VARCHAR(20) NOT NULL DEFAULT 'user'
   - Useful if upgrading from older database version
   
   ALTER TABLE users ADD COLUMN role VARCHAR(20) NOT NULL DEFAULT 'user' AFTER password
   
   getUserRole(mysqli $conn, string $email): string
   - Retrieves role for given email
   - Query: SELECT role FROM users WHERE email=?
   - Returns: 'admin' or 'user' (default)
   - Calls ensureUserRoleColumn() first
   
   isAdminUser(mysqli $conn, string $email): bool
   - Convenience function checking if user is admin
   - Returns true if role === 'admin'

4. USER REGISTRATION & LOGIN (register.php)
================================================================================

PURPOSE: Handles user sign-up, sign-in, and OTP verification initiation

KEY FUNCTIONS:

A. OTP EMAIL GENERATION:

   createLoginOtpHash(string $email, string $otp): string
   - Creates hash from email + "|" + otp
   - Uses hashToken() which calls sha256()
   - Example: createLoginOtpHash("user@email.com", "123456")
             = sha256("user@email.com|123456")
   - Stored in database for verification

   sendLoginOtpEmail(mysqli $conn, string $email, string $firstName, string $lastName): array
   
   PROCESS:
   1. Invalidates old unused OTPs for this email
      UPDATE login_verifications SET used_at=NOW() WHERE email=? AND used_at IS NULL
   
   2. Generates new 6-digit OTP (e.g., "000123" to "999999")
      otp = str_pad(random_int(0, 999999), 6, '0', STR_PAD_LEFT)
   
   3. Creates hash and inserts into database (tries 5 times in case of collision)
      INSERT INTO login_verifications (email, token_hash, expires_at)
      Token expires in 10 minutes: strtotime('+10 minutes')
   
   4. Sends email with formatted OTP and user's name
      Subject: "NBI Clearance Login OTP"
      Body: Shows OTP in large text (28px, bold)
   
   5. Returns array:
      [
         'success' => true/false,
         'error' => error message if failed,
         'delivery' => 'smtp' or 'file',
         'debug_otp' => actual OTP (if using file delivery for development)
      ]

B. USER SIGN-UP:

   Trigger: POST request with $_POST['signUp']
   
   VALIDATION CHECKS:
   1. Required fields: firstName, lastName, email, password
   2. Email must be valid format (filter_var FILTER_VALIDATE_EMAIL)
   3. Password minimum 8 characters
   4. Email must not already exist in database
   
   CODE FLOW:
   
   $firstName = trim($_POST['fName'] ?? '');     // Get & trim first name
   $lastName = trim($_POST['lName'] ?? '');      // Get & trim last name
   $email = trim($_POST['email'] ?? '');         // Get & trim email
   $password = $_POST['password'] ?? '';         // Get password (no trim)
   
   // Validate all fields present
   if ($firstName === '' || $lastName === '' || 
       !filter_var($email, FILTER_VALIDATE_EMAIL) || $password === '') {
       setFlashMessage('error', 'Please complete all sign up fields.');
       redirectTo('index.php?form=signup');
   }
   
   // Check password length
   if (strlen($password) < 8) {
       setFlashMessage('error', 'Password must be at least 8 characters long.');
       redirectTo('index.php?form=signup');
   }
   
   // Check if email already exists
   $checkStmt = $conn->prepare("SELECT email FROM users WHERE email=? LIMIT 1");
   $checkStmt->bind_param("s", $email);
   $checkStmt->execute();
   $checkResult = $checkStmt->get_result();
   $existingUser = $checkResult->fetch_assoc();
   $checkStmt->close();
   
   if ($existingUser) {
       setFlashMessage('error', 'Email address already exists.');
       redirectTo('index.php?form=signup');
   }
   
   // Hash password and insert new user
   $passwordHash = password_hash($password, PASSWORD_DEFAULT);  // bcrypt
   $role = 'user';  // Default role
   
   $insertStmt = $conn->prepare(
       "INSERT INTO users (firstName, lastName, email, password, role) 
        VALUES (?, ?, ?, ?, ?)"
   );
   $insertStmt->bind_param("sssss", $firstName, $lastName, $email, $passwordHash, $role);
   
   if ($insertStmt->execute()) {
       setFlashMessage('success', 'Account created successfully. Sign in to continue.');
       redirectTo('index.php');
   } else {
       setFlashMessage('error', 'Unable to create account. ' . $conn->error);
       redirectTo('index.php?form=signup');
   }

C. USER SIGN-IN:

   Trigger: POST request with $_POST['signIn']
   
   VALIDATION:
   1. Email must be valid format
   2. Password cannot be empty
   
   CODE FLOW:
   
   $email = trim($_POST['email'] ?? '');
   $password = $_POST['password'] ?? '';
   
   if (!filter_var($email, FILTER_VALIDATE_EMAIL) || $password === '') {
       setFlashMessage('error', 'Please enter your email and password.');
       redirectTo('index.php');
   }
   
   // Fetch user from database
   $stmt = $conn->prepare("SELECT email, password, firstName, lastName, role FROM users WHERE email=? LIMIT 1");
   $stmt->bind_param("s", $email);
   $stmt->execute();
   $result = $stmt->get_result();
   $user = $result->fetch_assoc();
   $stmt->close();
   
   // Verify password (handles MD5 to bcrypt upgrade)
   if (!$user || !verifyPasswordAndUpgrade($conn, $user['email'], $password, $user['password'])) {
       setFlashMessage('error', 'Not found, incorrect email or password.');
       redirectTo('index.php');
   }
   
   // Generate OTP and send email
   $mailResult = sendLoginOtpEmail($conn, $user['email'], 
                                   (string)($user['firstName'] ?? ''),
                                   (string)($user['lastName'] ?? ''));
   
   if (!$mailResult['success']) {
       // Email sending failed
       setFlashMessage('error', 'Unable to send OTP email. ' . $mailResult['error']);
       redirectTo('index.php');
   }
   
   // Store email in session for OTP verification page
   session_regenerate_id(true);  // Prevent session fixation
   $_SESSION['pending_login_email'] = $user['email'];
   
   // Store debug info if using file-based email
   if (($mailResult['delivery'] ?? '') === 'file') {
       $_SESSION['pending_verify_debug_code'] = (string)($mailResult['debug_otp'] ?? '');
       $_SESSION['pending_verify_debug_file'] = (string)($mailResult['debug_file'] ?? '');
   }
   
   redirectTo('verify_login.php');

5. LOGIN VERIFICATION (verify_login.php)
================================================================================

PURPOSE: OTP verification interface and logic

KEY VARIABLES:
- $pendingEmail: Email awaiting OTP verification (from session)
- $devVerifyCode: Debug OTP (shown in development/file mode)
- $devMailFile: Debug email file path

PROCESS: OTP VERIFICATION

Trigger: POST request with $_POST['verify']

FLOW:

1. Check pending email exists
   if ($pendingEmail === '') {
       $statusType = 'error';
       $statusMessage = 'No pending login was found. Please sign in again.';
   }

2. Extract 6-digit code from user input (remove non-digits)
   $otp = preg_replace('/\D+/', '', (string)($_POST['otp'] ?? ''));
   
   // Validate format
   if (!preg_match('/^\d{6}$/', $otp)) {
       $statusType = 'error';
       $statusMessage = 'Enter the 6-digit OTP code from your email.';
   }

3. Fetch OTP record from database
   Query: SELECT id, email, token_hash, expires_at
          FROM login_verifications
          WHERE email=? AND used_at IS NULL
          ORDER BY id DESC
          LIMIT 1
   
   - Checks email matches pending login
   - Only retrieves unused OTPs (used_at IS NULL)
   - Gets most recent OTP (ORDER BY id DESC)

4. Validate OTP not expired
   if (strtotime($record['expires_at']) <= time()) {
       // Mark as used to prevent reuse
       UPDATE login_verifications SET used_at=NOW() WHERE id=?
       
       $statusType = 'error';
       $statusMessage = 'Your OTP has expired. Please request a new code.';
   }

5. Verify OTP using hash comparison
   // Recreate hash from submitted OTP
   $expectedHash = createLoginOtpHash($pendingEmail, $otp);
   
   // Compare with stored hash using constant-time comparison
   if (!hash_equals($record['token_hash'], $expectedHash)) {
       $statusType = 'error';
       $statusMessage = 'The OTP code is incorrect. Try again.';
   }

6. Mark OTP as used (prevent replay attacks)
   UPDATE login_verifications SET used_at=NOW() WHERE id=? AND used_at IS NULL

7. Create authenticated session
   session_regenerate_id(true);  // Prevent session fixation
   $_SESSION['email'] = $record['email'];
   $_SESSION['role'] = getUserRole($conn, $record['email']);
   
   // Clear pending login data
   unset($_SESSION['pending_login_email']);
   unset($_SESSION['pending_verify_debug_code']);
   unset($_SESSION['pending_verify_debug_file']);

8. Redirect to dashboard
   if ($_SESSION['role'] === 'admin') {
       redirectTo('backend.php');
   }
   redirectTo('homepage.php');

SECURITY FEATURES:
- OTP never stored in plain text, only SHA256 hash
- hash_equals() prevents timing attacks
- 10-minute expiration prevents brute force
- Single-use tokens (marked used after verification)
- Session regeneration prevents session fixation
- Cannot reuse expired OTPs

RESEND OTP:

Trigger: POST request with $_POST['resend']

CODE:
if ($_SERVER['REQUEST_METHOD'] === 'POST' && isset($_POST['resend'])) {
    // Check pending email exists
    if ($pendingEmail === '') {
        $statusMessage = 'No pending login was found. Please sign in again.';
    } else {
        // Call sendLoginOtpEmail again with email
        $mailResult = sendLoginOtpEmail($conn, $pendingEmail);
        
        if ($mailResult['success']) {
            $statusType = 'success';
            $statusMessage = 'A new OTP has been sent to your email.';
            $devVerifyCode = ($mailResult['debug_otp'] ?? '');
            $devMailFile = ($mailResult['debug_file'] ?? '');
        }
    }
}

6. PASSWORD RECOVERY (forgot_password.php)
================================================================================

PURPOSE: Initiate password reset by sending email with reset link

PROCESS:

Trigger: POST request with email address

VALIDATION:
1. Email must be valid format (filter_var FILTER_VALIDATE_EMAIL)
2. User must exist in system (SELECT FROM users)

CODE FLOW:

1. Validate email format
   if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
       $statusType = 'error';
       $statusMessage = 'Please enter a valid email address.';
   }

2. Check if user exists
   $lookupStmt = $conn->prepare("SELECT firstName, lastName FROM users WHERE email=? LIMIT 1");
   $lookupStmt->bind_param("s", $email);
   $lookupStmt->execute();
   $user = $lookupStmt->get_result()->fetch_assoc();

3. If user exists:
   
   a. Invalidate old unused reset tokens
      UPDATE password_resets SET used_at=NOW() WHERE email=? AND used_at IS NULL
   
   b. Generate new reset token (32 bytes = 64 chars)
      $token = createToken();          // Random hex string
      $tokenHash = hashToken($token);  // SHA256 of token
      $expiresAt = date('Y-m-d H:i:s', strtotime('+30 minutes'));
   
   c. Store hash in database (NOT the plain token)
      INSERT INTO password_resets (email, token_hash, expires_at) 
      VALUES (?, ?, ?)
   
   d. Build reset link with plain token (hash not transmitted)
      $resetLink = appBaseUrl() . '/reset_password.php?token=' . urlencode($token)
   
   d. Send email with reset link
      Uses sendAppEmail() function (see mailer.php section)

4. Response (user never knows if email exists - security best practice)
   Shows same success message whether email found or not
   Prevents account enumeration attacks

DATABASE TABLE (password_resets):
- email VARCHAR(255)         - User's email
- token_hash VARCHAR(64)     - SHA256 of token
- expires_at TIMESTAMP       - 30 minutes from generation
- used_at TIMESTAMP NULL     - Set when used to prevent reuse
- created_at TIMESTAMP       - When created

7. PASSWORD RESET (reset_password.php)
================================================================================

PURPOSE: Handle password reset with token validation

INITIAL VALIDATION:

Token comes from URL: ?token=xxxxx

CODE:
$token = trim($_GET['token'] ?? $_POST['token'] ?? '');

if ($token === '') {
    $statusType = 'error';
    $statusMessage = 'The password reset link is invalid.';
} else {
    // Hash token to look up in database
    $tokenHash = hashToken($token);
    
    // Fetch reset record
    $tokenStmt = $conn->prepare(
        "SELECT id, email, expires_at, used_at FROM password_resets WHERE token_hash=? LIMIT 1"
    );
    $tokenStmt->bind_param("s", $tokenHash);
    $tokenStmt->execute();
    $tokenRecord = $tokenStmt->get_result()->fetch_assoc();
    
    // Check token validity
    if (!$tokenRecord) {
        $statusMessage = 'The password reset link is invalid.';
    } elseif (!empty($tokenRecord['used_at'])) {
        $statusMessage = 'This password reset link has already been used.';
    } elseif (strtotime($tokenRecord['expires_at']) <= time()) {
        $statusMessage = 'This password reset link has expired.';
    } else {
        $tokenValid = true;  // Proceed to form
    }
}

PASSWORD RESET FORM PROCESSING:

Trigger: POST request with new password

VALIDATION:
$newPassword = $_POST['password'] ?? '';
$confirmPassword = $_POST['confirm_password'] ?? '';

if (strlen($newPassword) < 8) {
    $statusMessage = 'Password must be at least 8 characters long.';
} elseif ($newPassword !== $confirmPassword) {
    $statusMessage = 'Passwords do not match.';
} else {
    // Hash password using bcrypt
    $newHash = password_hash($newPassword, PASSWORD_DEFAULT);
    
    // Update user password
    $updatePassStmt = $conn->prepare("UPDATE users SET password=? WHERE email=?");
    $updatePassStmt->bind_param("ss", $newHash, $email);
    $updatePassStmt->execute();
    
    // Mark token as used
    $consumeStmt = $conn->prepare("UPDATE password_resets SET used_at=NOW() WHERE id=?");
    $consumeStmt->bind_param("i", $tokenRecord['id']);
    $consumeStmt->execute();
    
    // Revoke all pending OTP login tokens for security
    $revokeLoginStmt = $conn->prepare(
        "UPDATE login_verifications SET used_at=NOW() WHERE email=? AND used_at IS NULL"
    );
    $revokeLoginStmt->bind_param("s", $email);
    $revokeLoginStmt->execute();
    
    // Redirect to login

SECURITY FEATURES:
- Token validated and hashed before display
- Single-use tokens (checked and marked as used)
- 30-minute expiration
- All pending logins revoked when password reset (forces re-login)
- Password validation requirements enforced

================================================================================

PURPOSE: Email delivery using PHPMailer library

FUNCTION: sendAppEmail(string $toEmail, string $toName, string $subject, string $htmlBody, string $altBody = ''): array

PARAMETERS:
- $toEmail: Recipient email address
- $toName: Recipient display name
- $subject: Email subject line
- $htmlBody: HTML email content
- $altBody: Plain text alternative (optional)

RETURN VALUE:
Array with keys:
- 'success': bool - Whether email was sent
- 'error': string - Error message if failed
- 'delivery': string - 'smtp' or 'file'
- 'debug_link': string - Main hyperlink from email (for file delivery)
- 'debug_file': string - Relative path to saved file (for file delivery)

DELIVERY MODES:

1. FILE MODE (Development/Testing):
   
   Trigger: config['driver'] === 'file'
   
   Process:
   - Logs email to HTML file instead of sending
   - File saved in mail_logs/ directory
   - Filename: YYYYMMDD_HHMMSS_randomhex.html
   - Includes metadata comments: timestamp, recipient, subject, primary link
   
   FILE CONTENT:
   <!--
   Time: 2026-03-22 14:30:45
   To: John Doe <john@example.com>
   Subject: NBI Clearance Login OTP
   Primary-Link: http://localhost/reset_password.php?token=xxxxx
   -->
   [HTML email content]
   
   Returns success=true with debug info
   Used to: Test emails without SMTP server, verify content

2. SMTP MODE (Production):
   
   Trigger: config['driver'] === 'smtp'
   
   VALIDATION:
   Checks required config keys:
   - host: SMTP server
   - username: SMTP username
   - password: SMTP password
   - port: SMTP port
   - from_email: Sender email
   
   SPECIAL HANDLING FOR GMAIL:
   
   if (contains(host, 'gmail.com')) {
       // Remove spaces and dashes from password
       password = str_replace([' ', '-'], '', rawPassword)
   }
   
   This fixes Gmail App Password format (16 chars, no spaces/dashes)
   
   SMTP CONFIGURATION:
   
   $mail = new PHPMailer(true);
   $mail->isSMTP();
   $mail->Host = $host;                    // SMTP server
   $mail->SMTPAuth = true;                 // Enable authentication
   $mail->Username = $username;            // SMTP username
   $mail->Password = $password;            // SMTP password
   $mail->Port = (int)$config['port'];    // SMTP port
   $mail->SMTPSecure = PHPMailer::ENCRYPTION_SMTPS  // For SSL
                      or PHPMailer::ENCRYPTION_STARTTLS  // For TLS
   
   EMAIL COMPOSITION:
   
   $mail->setFrom($fromEmail, $fromName);
   $mail->addAddress($toEmail, $toName);
   $mail->Subject = $subject;
   $mail->Body = $htmlBody;
   $mail->isHTML(true);
   if ($altBody) {
       $mail->AltBody = $altBody;  // Plain text fallback
   }
   
   SENDING:
   
   try {
       $mail->send();
       return ['success' => true, 'delivery' => 'smtp'];
   } catch (Exception $e) {
       return ['success' => false, 'error' => $e->getMessage()];
   }

smtpAuthHint(string $host, string $rawPassword): string

- Other: "Check SMTP host, port, encryption, username, and password."

================================================================================
                              SUMMARY OF DATA FLOW
================================================================================

REGISTRATION FLOW:
1. User → index.php (Sign Up form)
2. Form POST → register.php ($_POST['signUp'])
3. Validate fields and check email uniqueness
4. Hash password with password_hash()
5. Insert into users table
6. Flash message and redirect to index.php

SIGN IN FLOW:
1. User → index.php (Sign In form)
2. Form POST → register.php ($_POST['signIn'])
3. Validate email format and check password not empty
4. Query users table for matching email
5. Verify password with verifyPasswordAndUpgrade()
6. Call sendLoginOtpEmail()
7. Store email in session
8. Redirect to verify_login.php

OTP VERIFICATION FLOW:
1. User receives OTP email
2. User enters OTP on verify_login.php
3. Form POST → verify_login.php ($_POST['verify'])
4. Query login_verifications table for matching email
5. Validate OTP not expired
6. Hash user input and compare with stored hash
7. If match: Mark as used, create session, redirect to homepage
8. If not match or expired: Show error

PASSWORD RECOVERY FLOW:
1. User → forgot_password.php
2. Submits email address
3. Form POST → forgot_password.php
4. Generate random token (32 bytes)
5. Hash token with SHA256
6. Insert hash and expiration into password_resets table
7. Send email with plain token in URL
8. Show success (whether email found or not)

PASSWORD RESET FLOW:
1. User receives reset link with ?token=xxxxx
2. User → reset_password.php?token=xxxxx
3. Hash token and lookup in password_resets table
4. Validate token not used, not expired
5. Show password reset form if valid
6. User submits new password
7. Hash password with password_hash()
8. Update users table password
9. Mark token as used
10. Clear all pending OTP logins for account
11. Redirect to index.php with success message

ADMIN LOGIN FLOW:
1. User → admin_login.php
2. Submits email and password
3. Form POST → admin_login.php
4. Validate email format
5. Query users table for matching email
6. Verify password and check role='admin'
7. If valid: Create admin session, redirect to backend.php
8. If invalid: Show error, stay on page

USER SESSION FLOW:
1. User logged in ($_SESSION['email'] set, $_SESSION['role'] set)
2. User can access homepage.php and verify_login.php
3. User cannot access admin pages (backend.php, admin_login.php)
4. User clicks Logout → logout.php
5. Session destroyed
6. Redirect to index.php
7. User must login again

ADMIN SESSION FLOW:
1. Admin logged in ($_SESSION['email'] set, $_SESSION['role'] = 'admin')
2. Admin can access backend.php
3. Admin can view and manage:
   - Applications (nbi_applications table)
   - Appointments (nbi_appointments table)
   - Notifications (nbi_notifications table)
   - Users (users table)
4. Admin creates notifications which appear on user homepage
5. Admin updates application status
6. Admin schedules appointments
7. Admin can logout same as users

EMAIL FLOW:
1. User triggers email (signup fails, login OTP, password reset)
2. sendAppEmail() called with HTML and plain text versions
3. Config determines delivery mode:
   - FILE MODE: Saves HTML to mail_logs/ folder, returns debug link
   - SMTP MODE: Sends via Gmail SMTP, handles app password, returns success/error
4. Calling code handles returned array and displays message to user

                            SECURITY FEATURES SUMMARY

PASSWORD SECURITY:
✓ Bcrypt hashing (password_hash with PASSWORD_DEFAULT)
✓ Automatic MD5 to bcrypt migration
✓ 8-character minimum password length
✓ Confirmation field for password reset

SESSION SECURITY:
✓ Session regeneration on login (session_regenerate_id(true))
✓ Prevents session fixation attacks
✓ Session timeout on inactivity
✓ Session destroyed on logout

OTP SECURITY:
✓ Random 6-digit generation
✓ Hash-based storage (SHA256), never plain text
✓ Constant-time comparison (hash_equals)
✓ 10-minute expiration
✓ Single-use tokens (marked used after verification)
✓ Cannot disable expired OTPs from being used

TOKEN SECURITY:
✓ Cryptographically secure random generation (random_bytes)
✓ 64-character hex tokens (32 bytes)
✓ Hash-based storage (SHA256), never plain token stored
✓ 30-minute expiration for password reset
✓ Single-use tokens

ACCESS CONTROL:
✓ Role-based (admin vs user)
✓ Admin role verified from database (not just session)
✓ Protected pages check authentication and authorization
✓ Admin pages redirect to admin_login.php if not admin
✓ User pages redirect to login if not authenticated

SQL INJECTION PREVENTION:
✓ Prepared statements for all database queries
✓ Parameterized bound values (?
)
✓ Input validation before database operations
✓ Type-strict PHP with bind_param

TIMING ATTACK PREVENTION:
✓ hash_equals() for password verification
✓ hash_equals() for token comparison
✓ Constant-time string comparison

INPUT VALIDATION:
✓ Email format validation (filter_var FILTER_VALIDATE_EMAIL)
✓ Password length validation (minimum 8 characters)
✓ Required field checks
✓ Type casting and trimming ($_POST values)

OUTPUT ENCODING:
✓ htmlspecialchars() for all echoed user data
✓ Prevents XSS attacks
✓ Escaping in form values, messages, links

ERROR HANDLING:
✓ Errors logged to PHP error_log (not exposed to users)
✓ User-friendly error messages
✓ Generic "invalid credentials" message
✓ Prevents account enumeration (same message for email not found vs wrong password)

EMAIL SECURITY:

✓ App password for Gmail (not account password)

✓ Tokens in URLs expire quickly

ADMIN SECURITY:
